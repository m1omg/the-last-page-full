# The Last Page — Development Log

A retrospective on how **The Last Page** and its extended cut, **The Last Page —
Full Edition**, were built: the process, the tooling, the engineering decisions,
and the bugs worth remembering. For the original brief and the story bible, see
[ORIGIN.md](ORIGIN.md).

- **The Last Page** — https://m1omg.github.io/the-last-page/ · [repo](https://github.com/m1omg/the-last-page)
- **Full Edition** — https://m1omg.github.io/the-last-page-full/ · [repo](https://github.com/m1omg/the-last-page-full)

Both are original emotional JRPGs in the RPG-Maker/Omori lineage: a girl, a
shared sketchbook, a spreading ink stain, and an unfinished ending. Everything —
code, story, music, and the illustration prompts — was produced by Claude
(Opus 4.8 for the initial build, Fable 5 for the extension) through Claude Code,
with illustrations generated by GPT Image 2.

---

## 1. Two games at a glance

| | The Last Page | Full Edition |
|---|---|---|
| Length | ~30–45 min | ~2 hours |
| Drawn worlds | 3 (Meadow, Woods, Bay) | 5 (+ Eraser Dunes, If-Then Works) |
| Bosses (grief stages) | Anger, Sorrow, the Flinch, the Smudge | + Denial, Bargaining, and a secret superboss |
| Party | Mira, Sir Biscuit, Wisp | + Stub (the sun-yellow crayon) |
| Torn pages | 4 | 6 |
| Systems | emotion triangle, Reach Out | + charms, friend journal, side quests, the Mending chapter |
| Endings | 2 (wake / stay) | 2, reached after an optional revisit chapter |

The Full Edition is a fork of the original at its final commit, so every fix in
the base game carried forward. The two keep **separate save slots** and can be
installed side by side in one browser.

---

## 2. Ground rules

These constraints shaped every decision:

- **Compact & complete.** A finished story with real endings, not an open-ended
  demo. The Full Edition extends that promise rather than sprawling.
- **Fair contest.** The same brief was given independently to another agent tool
  (Codex). Neither implementation was ever allowed to look at the other. Codex
  was used here only as a *service* — the CLI that drives GPT Image 2 — never as
  a source to copy from.
- **Kindness is a mechanic.** Every enemy, every boss, can be resolved without
  violence. This is load-bearing, not decoration (see §7).
- **Save backward-compatibility is sacred.** Every change is checked against old
  saves before it ships. Saves are versioned, structurally validated on load,
  and migrated forward — never silently broken.
- **No build step, no runtime dependencies.** Vanilla JS + Canvas 2D, ES
  modules loaded directly. Audio is synthesized from scratch; there are no
  sampled assets. Illustrations are generated, then baked into the repo.

---

## 3. Architecture

- **Canvas** — 960×720 logical pixels, a 20×15 tile grid at 48px. One
  `requestAnimationFrame` loop; a `mode` string (`title` / `map` / `battle` /
  `cg` / `credits` / `gameover`) selects what updates and draws.
- **Scenes** — `MapScene` (overworld: painted background, hand-traced collision
  grid, entities, smooth grid movement, party followers), `BattleScene` (the
  turn engine), `Menu` (pockets/friends/journal/options), `Dialogue`
  (typewriter box, portraits, choices), and the cutscene interpreter.
- **State** — one plain object (`map`, `x/y/facing`, `flags`, `pages`, `party`,
  `inventory`, `journal`, …) persisted to `localStorage`. `version: 1`
  throughout; forward-compat is handled by migration, not version bumps (§8).
- **Data-driven content** — `maps.js`, `enemies.js`, `skills.js`, `items.js`,
  `party.js`, and `script.js` (the cutscene/dialogue tables) hold essentially
  all the content. The engine is small; the data is large.
- **Cutscenes** — a tiny interpreter over command objects (`say`, `choice`,
  `if`/`iflt`/`ifhas`, `flag`, `give`/`take`, `tp`, `fade`/`unfade`, `battle`,
  `page`, `cg`, `heal`, `save`, `ending`, …). The whole story is authored as
  data in this DSL.

## 4. The development loop

Every change went through the same pipeline, and nothing shipped until the last
step passed:

1. **Read the code.** Locate the exact function/data before touching anything.
2. **Change** the smallest surface that does the job.
3. **Simulate** balance changes with a Monte-Carlo model (§9) before trusting
   them.
4. **Headless-verify** behavior with real browser automation (Playwright): a
   full playthrough plus targeted probes that read live game state.
5. **Visually verify** by driving the real game to the moment and screenshotting
   it — then *looking* at the pixels, not just asserting on state.
6. **Deploy** to GitHub Pages.
7. **Byte-verify live.** `curl` the deployed files and confirm the bytes match;
   never trust the build API's "success."

Four standing verification layers back this up:

- `tools/validate.mjs` — static integrity: grid dimensions, reachability floods,
  no entity stranded in a wall, every referenced asset present, script commands
  well-formed, teleport targets walkable, gridSwaps may only *open* tiles.
- `tools/balance_sim.mjs` — a Monte-Carlo mirror of the battle formulas run over
  the real data files (§9).
- `tools/smoke.mjs` + `tools/touch_smoke.mjs` — headless full playthroughs (both
  endings) and the complete touch/mouse input matrix, via real multi-touch CDP
  events.
- `tools/collision_audit.{mjs,py}` — renders the collision grid as a translucent
  overlay on each painted background so art and walkability can be eyeballed
  together.

A **save-compatibility check** rides along on every change: a hand-crafted
pre-update save is loaded through the real `loadGame` path to confirm old saves
still open and migrate correctly.

## 5. Art pipeline (GPT Image 2)

- **Generation** — bespoke prompts in `tools/prompts.json` share a `style`
  prefix (children's-storybook crayon on cream paper) plus mode fragments
  (`topdown` for maps, `chroma` for sprites). Each is rendered by a `codex exec`
  call driving GPT Image 2, written to a gitignored `tools/_raw/` staging area.
- **Post-processing** — `tools/post_images.py` turns raw PNGs into shipping
  assets: opaque art (backgrounds, CGs, portraits) becomes JPEG; sprites are
  chroma-keyed (`#00FF00` → alpha) to PNG at fixed sizes.
- **Quota resilience** — Codex's free tier caps generations per day. Batches are
  written skip-if-exists so an interrupted run resumes cleanly across sessions.
  The Full Edition's 15 new illustrations were generated in one clean run with
  zero quota failures.
- **When generation over-reaches, patch locally.** GPT Image 2's *edit* mode
  will happily redraw a whole scene when asked for a small change. Three fixes
  this session were done with **PIL** instead: adding a wall-mounted TV to the
  apartment, erasing the fallen Swan and lighting the shrine candle in the risen
  Woods, and the Full Edition's twelve golden-hour "Mending" background variants
  (a channel-wise warm recolor). Local, deterministic, quota-free.

**Happy accident of the session:** asked for showcase renders of Mira holding
her sketchbook, GPT Image 2 spontaneously drew *both* Mira and Ren as crayon
doodles on the sketchbook's cover — three times running. It captured the story's
core image without being told to.

## 6. Audio pipeline

`tools/make_audio.py` is a pure-Python (no numpy) additive synthesizer. It
defines a handful of instruments — music box, felt piano, kalimba, soft square,
accordion, pads, sub-bass, filtered-noise percussion — and sequences original
melodies over chord tables, rendering seamless loops by wrapping note tails
around the buffer end. One recurring **friendship leitmotif** threads through
the title, endings, and boss themes, transposed and re-voiced per scene.

Two tracks were composed for the Full Edition, each scored to its world's grief
stage:

- **`bgm_dunes`** (denial) — a too-pleasant C-major lullaby with *holes* where
  the sad notes should land, and a faint, flat "ghost" voice a breath behind the
  melody: the song trying to un-remember itself.
- **`bgm_works`** (bargaining) — a clockwork escapement tick that never stops,
  kalimba gears, and a phrase that climbs toward resolution then *rewinds* four
  times — four better yesterdays — before finally being allowed to rest.

## 7. The battle system

The pacifist route is a real puzzle, not a mercy button.

- **Emotion triangle** — GIGGLY > GRUMPY > GLOOMY > GIGGLY. Advantage is 1.4×
  damage, disadvantage 0.75×, and each mood carries a passive quirk (giggly
  dodges; grumpy hits and takes more; gloomy tanks and regenerates ink).
- **The storm gate** — a doodle sitting in its distressed default mood *can't
  hear you*. The first good Reach Out breaks the storm (mood → neutral,
  "listening") but lands no heart. Only a listening doodle takes calm.
- **Settle & no-repeat** — at most one heart per enemy per round, and never the
  same comforting line twice in a row. Two party members can still say two
  *different* kind things in one round.
- **Kindness is armor** — every heart also softens that enemy's attacks. Calming
  a foe literally makes the fight safer.
- **Boss second wind** — a boss below half HP acts twice per round, announced
  once, so the back half of a boss fight has real teeth.
- **The cheer gamble** (added this session) — cheering an enemy GIGGLY lets it
  take *two* hearts a round, roughly halving the talk-down. The catch: a giggly
  doodle dodges and swings 15% harder, and a *failed* reach doesn't re-storm it —
  the giggle turns wild until the mood breaks. High risk, high friendship.

**Full Edition additions:**

- **Party of four** — a fourth combatant (Stub) required a re-laid battle UI
  (slimmer panels) and a full re-balance.
- **New skill kinds** — party-wide emotion shift (`emotion_party`) and an ink
  transfer (`inkgift`), Stub's support kit.
- **Charms** — one wearable per member, equipped from Pockets: start battles
  already listening, harden guard/wall, raise the heart cap, or trade defense
  for offense. Earned from side quests.
- **The rotating storm** — the secret superboss doesn't sit in one mood. Each
  heart you land *stirs up the next* bad feeling, so you must break through a
  fresh storm every round. It's the storm gate turned into a marathon.
- **Friend journal** — every soothed doodle is recorded; a *complete* journal is
  what unlocks the superboss, rewarding the pacifist route with the hardest and
  most personal fight in the game.

## 8. Bugs worth remembering (with root causes)

- **Boss-loss save corruption.** Boss scripts run `page`/`cg`/`tp`/`save`
  commands *after* the battle command, and a lost battle only resolves after the
  game-over screen is dismissed. Returning to title or continuing let the dying
  script's tail write into freshly reloaded state. **Fix:** `continueGame()` and
  `toTitle()` hold an `abortCutscenes` flag until `cutsceneDepth === 0`, so the
  zombie script fully unwinds before the mode flips. Regression-tested on both
  exits.
- **Credits freeze.** Pressing confirm on the credits nulled `game.credits`,
  then the draw path dereferenced it — killing the animation loop. **Fix:** null
  guards on the credits update/draw, plus a smoke step that presses confirm on
  the credits.
- **Torn-page stat stacking.** A world's `onEnter` could replay and re-grant its
  torn page, permanently buffing the party on every re-entry. **Fix:** the
  `page` command is idempotent (`if (inventory["page"+n]) return`), plus a scene
  gate and a regression test that re-enters the map and asserts max-HP is
  unchanged.
- **Flee loops.** Fleeing dropped the party adjacent to the doodle's home tile,
  which instantly re-triggered the fight. **Fix:** a 2.5s "flee grace" — chasers
  forget the party, bump-triggers are gated, and enemies flicker to signal the
  truce.
- **The dream ending played on a black screen.** The ending choice fades to
  black; the *true* ending un-fades after its teleport, but the *dream* ending
  script never did — so its entire closing montage and credits rendered under a
  full-black overlay, with only music audible. **Fix:** un-fade at the top of
  the dream-ending script, plus a smoke assertion that samples canvas luminance
  during the ending CG and fails if the frame is near-black.
- **Walkable painted walls.** Part of a solid-looking wall in the apartment was
  marked walkable, so Mira could stand *on* the door art. **Fix:** the alcoves
  are solid except a single doorstep tile under each painted door; arrivals land
  *on* the doorstep (spawning on a touch tile doesn't fire it — only completed
  moves do).
- **The missable fourth friend.** The Eraser Dunes shipped as a maze of
  one-tile corridors that didn't match the painted open sand, so players walked
  the only obvious path — straight to the boss — never meeting Stub or either
  wild doodle, while the boss cutscene still gave Stub dialogue as if he'd
  joined. **Fix:** the grid was opened where the art is soft (the bottom fan),
  putting both encounters inside chase range of the natural walk; and
  `s_dunes_boss` now has Stub skid in and join on the spot if `stub_joined` is
  still unset — a story-critical member must not be missable. Regression-tested
  in `tools/regress.mjs`, which drives the skipped-Stub path to the boss and
  asserts a four-member battle.
- **Self-narrating snacks & twin heal messages.** Using a snack on yourself
  read "Mira shares a Sugar Cookie with Mira", and Warm Glow / Candlelight
  printed the identical battle line. Both were single shared strings in
  `battle.js`; both now branch on the actual situation. And the Smoother's
  "Name the erased dog" option never named him — he's PATCH now, consistently,
  from the boss fight to the redraw scene.
- **The self-matching process footgun.** `pgrep -f smoke.mjs` and
  `pkill -f 'node tools/smoke.mjs'` match the *invoking shell's own command
  line*, spawning zombie watcher loops and once killing the controlling shell.
  **Lesson:** use the `ps … | grep '[s]moke'` bracket trick, or better, let the
  harness track background jobs.

## 9. Balance as measurement, not vibes

`tools/balance_sim.mjs` is a faithful Monte-Carlo mirror of the battle formulas
— damage, the emotion triangle, storm gate, settle/repeat rules, second wind,
calm softening, charms, and the rotating storm — run a few hundred times per
encounter against the *real* data files. It models a competent player on both
the fight and peace routes and reports median rounds, 90th-percentile rounds,
and loss rate. Critically, it also models torn-page growth (+8 HP / +4 ink each)
so pacing reflects the party you'd actually have at that point.

This turned "the boss dies too fast" into a number. Target bands: regular
enemies 2–4 rounds, first boss ~5, later bosses 6–8, and the Full Edition's
superboss ~11 rounds on the (intended) peace route — while brute-forcing it
loses ~62% of naive runs, exactly the "you were meant to talk it down" signal.
Every enemy stat retune was validated in the sim before it reached the game.

## 10. The Full Edition extension

The commission: *"make a longer, more fleshed-out version… kinda a machine
intelligence test."* Approach: **extend, don't sprawl** — keep Mira, Ren, the
Smudge, and both endings; give the timeline room to breathe.

- **Two new grief-stage worlds**, completing the set. The **Eraser Dunes**
  (denial): a desert of eraser shavings where doodles insist nothing's wrong,
  boss **THE SMOOTHER** — a serene giant with every doodle it ever erased
  ghosted inside it. The **If-Then Works** (bargaining): a clockwork theatre
  rewinding one street corner forever, boss **THE WIND-UP ORACLE**, who sells
  better yesterdays and has never once heard *no*.
- **Stub**, the fourth party member — Ren's worn-down sun-yellow crayon, made
  canon from an existing key item.
- **A playable failed hospital walk** mid-story: Mira's feet stop at the
  crossing and turn her home, which is what makes the true ending's walk cost
  something.
- **The Mending** — an optional post-Smudge chapter: revisit every world in
  healed, golden-hour light to say goodbyes and fight the secret superboss
  **THE UNFINISHED ONE** (the dragon Mira drew too big, got scared of, and never
  finished) before choosing an ending.

Engineering that made it clean:

- **`bgSwaps`** — a map can list flag-gated background overrides (first match
  wins), which drove both the risen Woods and the twelve Mending recolors.
- **`gridSwap`** — a flag can reshape a map's collision (validated to only ever
  *open* tiles, so it can't trap a mid-map save).
- **`ifjournal`** — a cutscene command that branches on a *complete* friend
  journal, gating the superboss.
- **`migrateSave`** — on load and on import, the union of each member's current
  skill list is folded in and missing sub-objects (like `journal`) are seeded.
  Old saves — including saves shaped like the *original* game's — open and play
  correctly. The whole extension shipped without a save-version bump.
- **Separate storage keys** so both games coexist in one browser.

## 12. Reproducing the work

```bash
npm run start                       # serve on http://localhost:4321
node tools/validate.mjs             # static integrity
node tools/smoke.mjs                # headless full playthrough (true ending)
SMOKE_ENDING=page node tools/smoke.mjs   # dream ending path
node tools/touch_smoke.mjs          # touch + mouse matrix
node tools/balance_sim.mjs          # Monte-Carlo pacing report
node tools/collision_audit.mjs && python3 tools/collision_audit.py
python3 tools/make_audio.py         # (re)synthesize all audio
python3 tools/post_images.py        # (re)bake images from tools/_raw
```

Add `?debug` to the URL for a collision overlay and `window.__game` hooks.

## 13. The pacing & kindness patch (2026-07-15)

A follow-up tuning session, all changes sim-checked against §9's bands:

- **+15% damage, both directions** — one multiplier at the top of `dmgTo`
  (and on the Keeper's bell, which bypasses it), mirrored in
  `tools/balance_sim.mjs`. Fights resolve about a round faster across the
  board; regular enemies still land in 2–4, later bosses 5–8, superboss ~10
  on the peace route.
- **The encore rule** — a GIGGLY doodle now accepts the *same* kind line twice
  in a row (`lastReach` is waived while giggly). Deepens the cheer gamble:
  giggly already took two hearts a round, now a lone party member can supply
  both.
- **Softening softened** — calm-softening is now `max(0.7, 1 − 0.045·calm)`
  (was `max(0.6, 1 − 0.06·calm)`): a doodle being talked down still stings,
  so the peace route carries slightly more risk. Peace losses stayed 0% in
  the sim.
- **Peace pays better** — a soothed doodle gifts its item ×2 (torn ones still
  drop ×1), and peace now restores +4 Ink alongside the +8 HP.
- **One new skill per member** — Mira's Quiet Word (sets an enemy `neutral`
  from the skill menu: a storm-break without burning a reach line — the
  `emotion` skill kind gained a neutral-aware message), Sir Biscuit's
  Emergency Biscuit (20 HP heal), Wisp's Dim the Lights (party-wide GLOOMY:
  damage reduction + ink regen as a defensive stance), Stub's Tiny Sun
  (enemy GIGGLY, the encore-rule enabler). Existing saves learn them via the
  `migrateSave` skill fold — no save-shape change, still `version: 1`.
- **One more good Reach Out option** on each early doodle and on Tangle, the
  Swan, the Keeper and the Smoother, so the no-repeat rule bites less on
  small rosters.
- **Woods shrine fixes** — the whimper text said "behind the shrine" while
  Wisp visibly hides in front of it; and the fallen queen was prompted as a
  "fallen giant origami crane" in `tools/prompts.json`, which is why she
  didn't read as a swan. Prompt rewritten; the baked `bg_woods_2` got an
  in-place GPT Image 2 edit (crane → swan) so the lit `bg_woods_2_risen`
  variant still matches.
